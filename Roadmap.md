## Phase 0：图内核 + 三视图骨架（PRD R1、R2）

**目标**：AI 一行代码都没有，但工作台已经是一个完整可用的工具（降级态即出生态）。

### 0.1 工程底座
- [ ] 仓库初始化：TS 全栈（Next.js）、monorepo 结构、开源 License 选定（AGPL？MIT？——见待决 D1）
- [ ] Docker Compose：Postgres + pgvector，一条命令起本地环境
- [ ] CI：lint + typecheck + 迁移测试

### 0.2 图内核（R1）
- [ ] 节点表：4 类型枚举（证据/需求/任务/结果）+ project scope 字段 + JSONB 正文 + 状态位（提议中/已确认）
- [ ] 边表：8 类型枚举（支撑/实现/验证/证伪/因为/取代/重复/阻塞）+ 风险级别标注 + 状态位
- [ ] 出处表：每条边关联（原始输入引用、创建者、置信度）——无出处的边写入被拒
- [ ] 确认记录 append-only：数据库触发器禁止 UPDATE/DELETE
- [ ] zod schema 与数据库枚举单一来源（防漂移）
- [ ] 验收测试：非法边类型写入报错；出处链完整可查；确认记录不可篡改

### 0.3 三视图（R2）
- [ ] 反馈收件箱：证据节点列表（创建/编辑/删除）
- [ ] 文档页：需求节点正文，TipTap/BlockNote 编辑器（**先做组件选型 spike，1 天封顶**）
- [ ] 看板：任务节点按状态列拖拽
- [ ] 手动连边 UI：任意视图内可给节点连 8 种边（选类型→选目标）
- [ ] 节点详情侧栏：任意节点可见其全部边与出处

### 🚪 P0 出口验收
- [ ] 纯手动完成：录 3 条证据 → 建 1 个需求（挂支撑边）→ 拆 2 张任务卡（挂实现边）→ 录 1 条结果（挂验证边）
- [ ] 任一节点修改即时反映到所有视图（同一数据源）

## Phase 1：Agent 运行时 + 捕获 + 审批 + 侧边栏（PRD R3、R5、R6）

**目标**：录入这件脏活从此归 AI；人只审不录。

### 1.1 Agent 运行时（架构 5.2/5.3）
- [ ] 事件层：Postgres LISTEN/NOTIFY + pg-boss 任务队列
- [ ] 统一五段流水线骨架：触发→上下文装配→LLM（Vercel AI SDK `generateObject`）→zod 校验→写提议
- [ ] 四份合同的配置结构（触发/权限/schema/模型）——先只实现捕获合同
- [ ] 数据库双凭证：读写凭证（捕获/推进用）与只读凭证（检索用，物理隔离）
- [ ] 模型接入：默认 API + provider 可换（本地模型选项留接口，不实现）

### 1.2 捕获 Agent（R3）
- [ ] 粘贴文本入口 → 提议态证据节点（去重、聚类）+ 提议边
- [ ] 文件上传入口（markdown/txt）
- [ ] 邮件转发地址（进食，不是客户端）
- [ ] 出处自动填充：每个提议节点/边带原始输入片段引用 + 置信度
- [ ] 预处理头接口设计（会议/反馈两个头先行，Slack/飞书头留到 P4）

### 1.3 审批（R5）
- [ ] 提议态视觉：全视图虚线呈现，未确认因果边对检索不可见（先落库约束）
- [ ] 审批 tab：批量 diff 视图（按天/按批次分组）
- [ ] 风险分级：低风险边（重复/阻塞/标签）自动生效+可撤销；高风险边（因为/验证/证伪/支撑）必须显式确认
- [ ] 接受/拒绝日志按边类型入库（v2 放权的原料，只记不用）

### 1.4 AI 侧边栏（R6）
- [ ] Cursor 式可开关侧边栏框架
- [ ] 锚定机制：跟随当前节点/页面，<1s 切换上下文
- [ ] 捕获入口 + 审批 tab 收纳进侧边栏
- [ ] 关闭态验证：无任何 AI 元素残留，P0 全功能可用

### 🚪 P1 出口验收
- [ ] 粘贴 20 条杂乱反馈 → 捕获产出聚类提议 → 审批 tab 5 分钟内批完 → 落图，全程键盘零录入
- [ ] 故意投喂垃圾输入：提议可整体拒绝，图无污染

## Phase 2：检索/追溯 + 冷启动导入（PRD R4、R7）

**目标**：产品的"哇时刻"上线——一键问出"当初为什么"。

### 2.1 语义索引
- [ ] Embedding 管线：节点创建/更新时自动向量化（pgvector）
- [ ] Project scope 过滤 + 升级式检索（局部无果→查全局并标注来源项目）

### 2.2 检索 Agent + 追溯搜索栏（R4）
- [ ] 两段式检索：向量定位入口 → 沿类型边遍历（纯 SQL，零幻觉）
- [ ] 生成层硬约束：仅可引用遍历到的节点；每句话挂深链；无引用陈述直接拦截
- [ ] "图里没有记录"路径（明确拒答，不推测）
- [ ] Cmd-K 全局追溯搜索栏
- [ ] 检索评估集 v1：≥20 个问答对（含 5 个"应拒答"用例），CI 跑精确率

### 2.3 冷启动导入（R7）
- [ ] 通用 Markdown/CSV 导入 → 走捕获管线熔图
- [ ] Notion 导入器（导出包解析 → 预处理头）
- [ ] 飞书导入器
- [ ] 大批量并行：同一运行时多 worker（5000 条级别压测）

### 🚪 P2 出口验收 = Demo 故事线完整可演
- [ ] ① 导入真实历史（Notion/飞书导出）→ ② 审批确认因果边（5 分钟）→ ③ Cmd-K 提问"X 当初为什么"→ ④ 回答每句可点击直达节点 → ⑤ 关侧边栏，工作台完整可用
- [ ] 检索评估集精确率达标（错误引用 0 容忍）

## Phase 3：Dogfooding（用工作台管理工作台自身）

**目标**：本产品自己成为图里的第一个项目；用真实使用回答"这东西到底有没有用"。

- [ ] 把《PRD-MVP.md》熔成图中第一个需求节点树（四份 md 文件全部导入）
- [ ] 本 Roadmap 的每个勾选项变成图中的任务节点
- [ ] Leading 指标埋点：捕获接受率 / 高风险边错连率 / 引用点击率 / 拒答占比 / 审批耗时
- [ ] 简易指标面板（内部用，够看即可）
- [ ] 每周复盘：新问题记录为图中证据节点（吃自己的狗粮，闭环自己的环）

### 🚪 P3 出口验收（4 周后）
- [ ] 捕获接受率 ≥70%；高风险边错连率 <5%；周追溯查询 ≥10 次
- [ ] Go/No-Go 决策：指标达标 → 找首个设计伙伴团队；不达标 → 定位是捕获质量、检索质量还是审批疲劳，修完再评
