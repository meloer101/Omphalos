# AI-native PM 工作台 —— 开源选型表

> 核实时间：2026-07（见各行来源）。License 判断前提：目标用户为「公司内部产品团队」，产品形态为自建核心 + 借鉴/集成开源件，非直接打包转售第三方 AGPL 代码。

> **⚠️ 角色变更（2026-07-05，见《Agent架构设计.md》第〇节）**：架构已确定"工具即 body"——所有工具原生长在自己代码库、共用一个数据库，**不走 MCP/API 粘合路线**。本表中项目的用途从"集成对象"变为**"解剖对象"**：抄它们的数据模型和交互思路，不接它们的 API。仅两个例外仍需连接器：冷启动历史导入（Jira/Notion/Slack/飞书 → 图）和指标数据源（埋点 → 结果节点），且只存在于单向进食边界。若直接嵌入开源代码，组件优先选 MIT（编辑器如 TipTap/BlockNote 等）。

## 二、结论与选型原则

1. **目标用户是"公司内部产品团队"，非直接转售开源代码本体** → AGPL（Plane / Metabase / Formbricks / Docmost）的风险大幅降低，可放心借鉴其思路与自建时的架构参考。
2. **唯一需要明确排除的：n8n。** 它不是真正开源（fair-code / Sustainable Use License），"托管并收费"正是其许可证明确禁止的场景。工作流编排层建议自建，或参考 Activepieces（MIT）思路。
3. **Penpot 是所有候选中商业化最友好、AI 集成最天然的一个**（MPL-2.0 + 官方 MCP Server），设计环节可优先深入调研。
4. **AppFlowy / Affine 的"开源"有水分**（多人协作要收费/限流），文档环节若要自建对标，Docmost 的模型更值得参考（无人数限制，纯 AGPL）。
